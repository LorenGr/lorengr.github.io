Overview

AngularFlatStrap is an evolving collection of components that can be used within your AngularJS application.

Dependencies

  • AngularJS Tested on version 1.3
  • FlatStrap CSS ('Flat' design theme)
  • Angular-UI Bootstrap - Buttons (ui.bootstrap.buttons) *only required if checkbox type cells are used. Tested on version 0.10.0
  • jQuery DateTimePicker plugin *only required if datetime type cells are used. Tested on version 2.1.9

Files to download

Build files for all directives are distributed in a main single file called build/angular-flatstrap.js (minified for production usage). This file contains all the directives and templates you will need to impliment the components in your app. If you only wish to include individual components, you can find the build files in the folder /src/[component name]/build/[component name].min.js

Installation

Install AngularFlatStrap

As soon as you've got all the files downloaded and included in your page you just need to declare a dependency on the flatstrap module:

{% highlight text %} angular.module("myModule",["flatstrap"]) {% endhighlight %}

Bower Install

{% highlight text %}$ bower install angular-flatstrap{% endhighlight %}

FlatGrid (flatstrap.flatgrid)

FlatGrid is a data grid component for your AngularJS application. FlatGrid can be setup through 1 object which contains the row data, the column setup and the grid general options. Some of the features that make this grid special are :

  • Excel-like editing and cell navigation
  • Cells can contain not only text, but various types of interactive form elements.
  • Add, delete and edit grid contents
  • Slick 'Flat' style look and feel
  • Pagination Feature
  • Search box to find the exact row that you need
  • and more improvements to come...

Configuration Object

    rows : An Object Array containing all the rows of the grid.
    columns : An Object Array which will determine the FlatGrid column setup (object per column).
    • Name : The Name of the object that this column should retrieve its contents from.
    • Label : The title text to show on the column header
    • Editable (Default: true) : Boolean that toggles cell readonly
    • Sortable (Default: true) : Boolean that enables/disables column sorting through the column header
    • AutoFocus (Default: false) : Boolean that sets focus (onload), to the postForm* field of this column
    • Form (Default: "input") : The object type inside each cell of the column.
      • inputText Field
      • checkboxRadio button (toggler) *requires ui.bootstrap.buttons
      • datetimeDate and Time popup selector *use UTC time format in source json
      • buttonA clickable button with an optional callback
    options : Parameters which will be applied to all the grid.
    • title (Default: null) : A title header for the entire grid.
    • defaultSort (Default: First Column) : The name of the object representing the column to sort.
    • defaultSortOrder (Default: 'asc') : Can be 'asc' or 'desc'
    • controls (Default: true) : Controls appear on the last column of each row.They serve to edit/delete.
    • postForm (Default: true) : Show or hide the postForm
    • pagination (Default: true) : When false, grid becomes scrollable.
    • search (Default: true) : Shows or hide row searching text field.
      Events : function callbacks
      - Callbacks need to be accessible through scope
      - Row data is available as callback first argument. ([Argument Name].data contains raw data).
      - Callbacks must return resolved promises
      • onrowfocus : If a callback is provided grid rows become clickable.
      • onformbuttonclick : triggered when the 'button' inside a row cell is clicked
      • ondelete : triggered when a row is deleted through the row 'controls'
      • onadd : triggered when a new item is added through the postForm*
      • onsave : triggered when a row is modified and confirmed
* postForm is the first row of the grid that serves to add a new row
